# Exploratory Scripts

This directory contains intermediate scripts and notes that shaped the final protocol but are not part of the main pipeline. They are kept for process transparency and reproducibility; none of them are referenced by the report, by `RESULTS.md`, or by any active production script.

## Scripts

| File | Purpose | Why archived |
|---|---|---|
| `evaluate_baseline.py` | Runs only the rule-based baseline on `data/final/test.csv` and prints the metrics. | Superseded by `evaluate_all.py`, which evaluates the baseline alongside the three neural systems. |
| `train_extended_multi_seed.py` | Early 5-seed variance study of Extended BERT alone. | Superseded by `train_full_benchmark.py`, which sweeps 20 seeds across all three neural systems. |
| `train_bert_roberta_multi_seed.py` | Early 5-seed variance study of Standard BERT and RoBERTa. | Same reason — `train_full_benchmark.py` replaces both. |
| `train_extended_continue.py` | Loads the best Extended BERT checkpoint and trains it for 5 additional epochs to diagnose overfitting. | Conclusion (val F1 drops while train loss falls) is now read directly from the 10-epoch learning curves generated by `train_ext_roberta_10ep_curves.py`. |
| `train_roberta_improved.py` | Re-trains RoBERTa with a tuned hyperparameter bundle (lr = 1e-5, weight decay = 0.01, 10 epochs) to test whether gentler training improves generalisation. | Empirical gain was small and not statistically significant; the main report uses the default RoBERTa recipe. Retained as a sensitivity-analysis record. |
| `make_roberta_curves.py` | Plots the RoBERTa-only hyperparameter-tuning comparison (fig7). | Fig7 is not cited by the final report. |
| `PARTA_REPORT.md` | Original markdown draft of the Part A / data-chapter write-up. | Content absorbed and rewritten in the final LaTeX report (`conference_101719.tex`). Kept for drafting history. |

## How to Re-run

Each script is still self-contained and runnable with `python experiments/<script>.py` from the project root. The scripts write to the same `outputs/evaluation/` directories as the main pipeline, so re-running them after the main pipeline is complete will not corrupt the headline results.
